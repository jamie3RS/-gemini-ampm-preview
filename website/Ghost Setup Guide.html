Step 02

Pack and upload the theme

The theme I've built lives in this project as a folder called ampm-mothership/. Ghost expects themes as a single .zip file.

On your computer

  1. Download the ampm-mothership/ folder from this project.
  2. Open the folder.
  3. Select all files inside the folder (not the folder itself).
  4. Right-click → Compress (Mac) or Send to → Compressed folder (Windows). This produces ampm-mothership.zip.
Common trap

Make sure the zip contains package.json at its root — not inside a nested ampm-mothership/ folder. Ghost rejects themes where package.json is nested.

In Ghost Admin

  1. Log into Ghost Admin at your-site.ghost.io/ghost/.
  2. Go to Settings → Design & branding → Custom theme → Change theme.
  3. Click Upload theme, drag ampm-mothership.zip in.
  4. Once uploaded, click Activate.

You should now see the AMPM design when you visit your Ghost site's URL — but most pages will look broken or empty until you complete the next steps. That's expected.

Step 03

Upload the routes file

Ghost's URL structure is configured via routes.yaml. The theme ships with one — it lives in the same ampm-mothership/ folder. This step wires /work/, /insights/, /companies/ etc. to the right templates.

  1. In Ghost Admin → Settings → Labs → Beta features.
  2. Scroll to Routes → click Upload routes file.
  3. Select routes.yaml from your downloaded theme folder.
If you see "routes upload failed"

It usually means the routes file is referencing a template that doesn't exist yet — which it shouldn't, because the theme contains them all. If this happens, screenshot the error and message me; it's a 30-second fix.

Step 06

Add your first content — paste the seed

The content-seed/ folder in this project has 4 ready-to-publish posts:

  • case-study-01-london-hospital.md — Gemini AMPM, healthcare, passive fire
  • case-study-02-mayfair-fitout.md — AMPM Building Services, office fit-out
  • insight-01-ppn-eot.md — Jamie's piece on EOT social value
  • insight-02-bsa-golden-thread.md — Dan's piece on BSA

Each markdown file starts with a frontmatter-style block telling you what to set in the Ghost editor (tags, excerpt, author). Below that is the body — paste it into Ghost's markdown editor.

Steps per post

  1. Ghost Admin → Posts → New post.
  2. Copy the title from the markdown file's H1.
  3. Click the cog → set the tags, custom excerpt, author, and feature image as the frontmatter says.
  4. In the body, switch the editor to Markdown mode (or use Ghost's slash-command to insert a Markdown card), then paste the body content.
  5. Click Publish when ready. For internal-only review, you can leave them as Drafts for now — they'll only show on the live /work/ or /insights/ pages once published.
Authors

The first time you publish under "Dan Small" or "Luke Finney", you'll need to create them as Staff in Ghost Admin → Settings → Staff → Invite people. Give them an Author role (lowest permission); they don't need full admin. Set their Bio to the line that should appear under their byline (e.g. "Managing Director, AMPM Group").

Step 07

Connect your domain — staging subdomain

While you're internally reviewing, point a subdomain of ampm.co.uk at Ghost — not the apex domain. staging.ampm.co.uk or preview.ampm.co.uk are sensible.

In Ghost Admin

  1. Open the support widget (bottom right) → request "Custom domain" → tell them staging.ampm.co.uk.
  2. Ghost(Pro) will email you a CNAME target — something like your-site.ghostflex.com.

In your domain registrar

  1. Log into wherever ampm.co.uk is registered (GoDaddy, 123-Reg, Cloudflare, etc).
  2. DNS settings → add a new CNAME record.
  3. Host / Name: staging (just the subdomain part, not the full domain)
  4. Value / Target: the CNAME target Ghost gave you.
  5. TTL: leave default (3600 / 1 hour).
  6. Save.

DNS takes 5–60 minutes to propagate. Ghost(Pro) will auto-provision an SSL certificate as soon as the CNAME resolves — usually within 30 minutes of the DNS going live. Once provisioned, https://staging.ampm.co.uk shows your site.

Step 08

Lock with a password until launch

While internal review is happening, you don't want the site indexed by Google or stumbled on. Two options, in order of friction:

Option A — Ghost private mode

Built-in to Ghost. Single password, applied to the whole site. Search engines blocked. Good for internal review.

Set up: Settings → General → Make this site private → set password. Anyone visiting will see a password gate; once they enter it, they see the site normally.

Free · included

Option B — Cloudflare Access

Allow-list specific emails (e.g. @ampm.co.uk). Each reviewer logs in with Google or a one-time email link. Better for showing trustees + advisors without sharing a password.

Set up: needs Cloudflare in front of your domain — 20-minute setup. I'll walk you through if you want this.

Free · up to 50 users

I'd start with Option A for the first review pass — single password is fine for a small internal group — and move to Cloudflare Access closer to launch.

Step 09

Going live

Once internal review is done and the content reads right:

  1. Move the domain. Repeat step 7 but with ampm.co.uk (apex) or www.ampm.co.uk. Most domains require an A record for apex; Ghost(Pro) gives you the IP.
  2. Disable private mode. Settings → General → uncheck Make this site private.
  3. Submit your sitemap to Google. Ghost generates /sitemap.xml automatically. Submit it via Google Search Console so the site starts ranking.
  4. Set up Members & newsletter. Settings → Membership → enable. The newsletter signup form on the site (in the footer and at the bottom of insights) starts working immediately. Ghost handles the welcome email and unsubscribe link natively.
  5. Analytics. Either Plausible (GDPR-clean, paid) or Fathom (similar). Drop the snippet into Settings → Code injection → Site header. Avoid Google Analytics if you can — its cookie consent banner is a bad first impression on a B2B site.
Step 10

Ongoing — how we work together

Now the site is live, the working pattern is:

For content (case studies, insights, news)

  1. You send me the raw material — notes, transcripts, bullet points, a half-written draft.
  2. I produce a finished post in your voice, formatted as markdown the way the seed posts are.
  3. You paste it into Ghost. Add the feature image, tags, author, custom excerpt.
  4. You publish — or save as draft for the next review pass.

For design changes (new page types, layout tweaks, brand applications)

  1. You describe what you want. Inline-comment any element on the prototype if it helps.
  2. I prototype the change in this design sandbox first, you sign off.
  3. I update the relevant Ghost theme file (.hbs or .css).
  4. You repack the ampm-mothership.zip and upload it to Ghost Admin → Design.
  5. Site updates within seconds.

For optimisation (SEO, conversion, growth)

  1. Periodically share your Search Console + Plausible dashboards.
  2. I'll identify what's working, what's not, and what to publish next.
  3. We adjust the editorial calendar accordingly.

Then I commit directly to the repo and your site updates without you uploading anything. It's a 1-hour setup and removes the upload step from the loop forever. Ask me when you're ready.
